SDL_AudioFormat

作用

音频格式的枚举。

Audio Format Macros

SDL_AUDIO_MASK_BITSIZE (0xFF) 它帮助你从音频格式中提取出位深度(bit size)。在处理音频数据时，了解音频格式的位深度是很重要的，因为它决定了声音的质量和数据的大小。位深度越高，能表现的声音细节越多，但相应的数据量也越大。
SDL_AUDIO_MASK_DATATYPE (1<<8) SDL_AUDIO_MASK_DATATYPE通常与音频格式相关的宏一起使用，以确定音频数据是有符号还是无符号类型，以及它是8位、16位、32位还是浮点数。这个宏可以帮助开发者在处理不同的音频数据流时做出适当的处理决策。
SDL_AUDIO_MASK_ENDIAN (1<<12) 用于表示音频数据的字节序。在处理音频数据时，不同的平台可能会使用大端（Big-Endian）或小端（Little-Endian）的方式来存储数据。这个宏可以帮助开发者识别当前平台的字节序，并据此进行相应的处理。
SDL_AUDIO_MASK_SIGNED (1<<15) SDL_AUDIO_MASK_SIGNED 宏用来判断音频数据是否是有符号的。当你获取到 SDL 音频格式时，可以通过与这个宏进行位操作来检查音频样本是否是有符号的。
SDL_AUDIO_BITSIZE(x) (x & SDL_AUDIO_MASK_BITSIZE) SDL_AUDIO_BITSIZE 是一个宏，用于从SDL音频格式标志中提取音频样本的位大小。当您使用SDL库进行音频处理时，您可能会遇到包含SDL_AUDIO_BITSIZE的音频格式标志。这些标志定义了音频数据的具体格式，包括样本的位深度（如8位、16位等），是否有符号以及是否为大端字节序。
SDL_AUDIO_ISFLOAT(x) (x & SDL_AUDIO_MASK_DATATYPE) 用于检查SDL（Simple DirectMedia Layer）库中的音频格式是否为浮点格式。使用 SDL_AUDIO_ISFLOAT 宏可以帮助开发者确定SDL_AudioFormat变量是否表示浮点数据。如果音频格式是浮点的，该宏将返回非零值；否则返回零。这在处理音频流时非常有用，因为你可能需要根据数据类型采取不同的处理方法。
SDL_AUDIO_ISBIGENDIAN(x) (x & SDL_AUDIO_MASK_ENDIAN) 在处理音频数据时，不同的系统可能会采用不同的字节序。字节序是指多字节数据的存储顺序问题，主要有大端（Big-endian）和小端（Little-endian）两种格式。大端模式下，数据的高位字节存储在内存的低地址端，而小端模式下，数据的高位字节存储在内存的高地址端。
SDL_AUDIO_ISSIGNED(x) (x & SDL_AUDIO_MASK_SIGNED) 用于确定音频样本是否为有符号的。
SDL_AUDIO_ISINT(x) (!SDL_AUDIO_ISFLOAT(x)) SDL_AUDIO_ISINT宏可以用于判断该结构中的音频格式是否为整数格式。整数格式是指采样值以整数形式表示，而不是浮点数形式。
SDL_AUDIO_ISLITTLEENDIAN(x) (!SDL_AUDIO_ISBIGENDIAN(x)) 它用于在SDL（Simple DirectMedia Layer）库中确定当前音频系统的字节序是否为小端模式。
SDL_AUDIO_ISUNSIGNED(x) (!SDL_AUDIO_ISSIGNED(x)) 宏通常接受一个音频格式参数，并返回一个布尔值，指示该格式是否为无符号类型。

Audio Format Values

8-bit support
AUDIO_S8 signed 8-bit samples
AUDIO_U8 unsigned 8-bit samples
16-bit support
AUDIO_S16LSB signed 16-bit samples in little-endian byte order
AUDIO_S16MSB signed 16-bit samples in big-endian byte order
AUDIO_S16SYS signed 16-bit samples in native byte order
AUDIO_S16 AUDIO_S16LSB
AUDIO_U16LSB unsigned 16-bit samples in little-endian byte order
AUDIO_U16MSB unsigned 16-bit samples in big-endian byte order
AUDIO_U16SYS unsigned 16-bit samples in native byte order
AUDIO_U16 AUDIO_U16LSB
32-bit support (new to SDL 2.0)
AUDIO_S32LSB 32-bit integer samples in little-endian byte order
AUDIO_S32MSB 32-bit integer samples in big-endian byte order
AUDIO_S32SYS 32-bit integer samples in native byte order
AUDIO_S32 AUDIO_S32LSB
float support (new to SDL 2.0)
AUDIO_F32LSB 32-bit floating point samples in little-endian byte order
AUDIO_F32MSB 32-bit floating point samples in big-endian byte order
AUDIO_F32SYS 32-bit floating point samples in native byte order
AUDIO_F32 AUDIO_F32LSB
