在写本节之前还是简单介绍一下如何装扩展库SDL_ttf，安装过程和之前的SDL_image相差不大。（IDE使用的是CB，其他IDE请读者自行搜索）。

Add SDL_ttf to CB

SDL_ttf下载地址：click to jump
下载完之后博主将其更名为mingw_dev_lib。


选择项目的名称 -> Properties -> Project’s build options



注意这里需要选择左上角的SDL2(项目名)，确保接下来的设置能同时涉及到Debug和Release。选择菜单中的 Search directiories -> Complier， 将mingw_dev_lib文件夹下的include中的SDL2文件路径加入进来。


选择菜单中的Linker将mingw_dev_lib中的lib文件路径添加到其中。


选择菜单栏中的Linker Setting，在-lSDL2_image之后添加-lSDL2_ttf。需要注意的是其中l是L的小写，而不是i的大写。


最后从mingw_dev_lib文件夹中的bin文件夹中将SDL2_ttf.dll文件复制到项目的工作目录当中。

在分析本节代码之前，有一些背景知识需要提前了解一下。首先什么是ttf。根据wikipedia的描述，ttf是一种类型字体文件的扩展名，而这种字体被称为TrueTyoe，是微软和苹果开发的一种电脑轮廓字体的标准类型。具体这里不详细介绍，感兴趣的可以自行进入连接进行阅读。

TTF_FRONT

TTF_FRONT是SDL2为了描述ttf而创建的一种数据结构，其是SDL_ttf库的一部分。一般的使用的方法是通过SDL_ttf加载TrueType字体文件，然后通过SDL的其他功能进行文本的渲染。

True Type Fonts

相对于之前的代码，首先不同的是头文件的引用，除了上面教程中的将SDL_ttf导入IDE的操作之外，不要忘了在代码中将使用include将SDL扩展库导入。

相比于之前的LTexture定义，多了类函数loadFromRenderedText,此函数的作用简单概括来说就是将从文件中读取到的ttf首先通过SDLDictionary TTF_RenderText_Solid函数转换为Surface格式，然后通过SDLDictionary SDL_CreateTextureFromSurface转换成Texture格式，方便之后的渲染处理。

在声明全局变量的时候增加了一个TTF_Font的声明，如果仔细观察的话就能发现在全局变量声明的时候忽略了Surface和Texture。首先对于Surface只是作为过渡到Texture的临时变量，所以作用域放在类函数中足矣。而对于Texture代码采用了类来处理，便无需再另外声明全局Texture, 取而代之的是全局类。

对于TTF_Front也样同样需要初始化，其初始化过程是通过函数TTF_Init()完成的，但相比于其它的初始化，优先级是最低的，所以对于TTF_Front的初始化放在了init()函数的最后。

对于loadmedia函数而言，它的任务从之前的博客到现在总是从指定的文件路径读取对应的文件。而这次我们需要的是ttf文件，所以在函数中读取函数替换为SDLDictionary TTF_OpenFont，在读取ttf文件的同时设置字体的大小。对于之后字体的前置颜色的设定则使用了SDLDictionary SDL_Color。在本例中为0，0，0，根据RGB表，所选的颜色为黑色。

同之前的全局变量一样，TTF_Font同样在使用完之后需要释放对应的资源。这里有一点需要注意：为了释放之前的TTF_Font资源首先需要使用SDLDictionary TTF_CloseFont丢弃创建的字体，然后才是SDLDictionary TTF_Quit来取消SDL_ttf的初始化。
