处理数据

3.1. 简单变量

3.1.1. 变量名

  • 变量命名规则：名称中只使用字母字符，下划线，和数字
  • 名称第一个不能是数字
  • 区分大写和小写
  • 不能使用C++关键字
  • 以两个下划线或下划线或大写字母打头的留给编译器及其使用的资源使用
  • C++对名称长度没有限制

3.1.2. 整型

  • C++的基本整型包括char, short, int, long和C++11新加的long long

3.1.3. short, int, long和long long

  • 通过使用不同的位bit来存储值。
  • Short至少16位
  • Int 至少与short一样长
  • Long 至少32位，且至少与int一样长
  • Long long 至少64位，且至少与long一样长
  • 4中类型都属于符号类型

3.1.4. 无符号类型

  • Unsigned short
  • Unsigned int
  • Unsigned long
  • Unsigned long long
  • C++不保证符号整型超越限制，发生上溢和下溢。

3.1.5. 整型字面值

  • 整型字面值（常量）是显示地书写常量，C++能以三种计数方式书写：10基数（如93），8基数（如042），16基数（如0x42）。
  • Cout提供dec, hex, 和oct分别用于指示cout以十进制，十六进制和八进制显示整数。

3.1.6. C++如何确定常量地类型

  • 除非使用了特定地后缀或值太大，C++将整数常量存储为int类型
  • L或l后缀表示long
  • U或u表示unsigned int
  • ul表示unsigned long
  • ll或LL后缀表示long long
  • ull后缀表示unsigned long long

3.1.7. char类型：字符和小整数

  • 很多系统支持的字符都不超过128个
  • 在使用char类型时cin和cout会自动将字母和ASCII之间互相转换。
  • C++对字符使用单引号，对字符串使用双引号

3.1.8. char字面值

  • 转义序列：当有些字符不能直接通过键盘输入到程序之中时。
  • 可以使用八进制和十六进制编码来使用转移序列。将其单独放在单引号之中或一同放进双引号之中。

3.1.9. 通用字符名

  • 通用字符名类似于转移序列，可以以\u或\U开头，对于前面那个，后面接的时16进制位，对于后面那个接的是8进制位。
  • Unicode提供了一种表示各种字符集的解决方案，位大量字符和符号提供标准数值编码，并依据类型进行分组。

3.1.10. signed char和unsigened char

  • 在默认情况下，char既能有符号也能没有符号，这与具体的C++实现有关。但总有一些情况需要确定char的符号类型，其中unsigned char的范围在0255之间而singed char的范围在-128127之间。

3.1.11. wchar_t

  • 对于字符集大于8位字节的，C++的处理方式有两种：编译器直接将char定义为一个16位的字节或更长，使用wchar_t来表示扩展字符集。在一个系统中，它的底层类型选择取决于实现，可能是unsigned short或者int。
  • 如果要使用iostream来处理wchar_t流，需要使用到wcin和wcout。并且可以追加前缀L来指示宽字符常量和宽字符串。

3.1.12. C++11新加的char16_t和char32_t

  • 在字符串编码时，如果能有特定长度和符号特征类型，而不是wchar_t这种随实现而变化的类型。Char16_t和char32_t便应运而生，都属于无符号。前者使用前缀u而后者使用前缀U。
  • 对于通用字符名使用相同的规则。

3.1.13. bool 类型

  • 在计算中，布尔值可能位true或false。
  • C++的非零值解释为true，零值解释位false。

const限定符

  • 为了避免在修改常量时修改多处，C++提供了一种更好的方式来修改变量声明和初始化，使用const。
  • 对于常量一般名称首字母大写。
  • 在声明常量时需要同时进行初始化。
  • 对比define，const能指定类型，其次能将作用域规则限定在特定的函数或者文件之中，最后const能用于更复杂的类型。

浮点数

  • 如果数字很大无法使用long来表示，可以使用浮点数。

3.3.1. 书写浮点数

  • 标准小数点写法
  • E表示法，如3.45E6(注意中间不能有空格)

3.3.2. 浮点类型

  • 存在三种浮点类型：float，double和long double。
  • 有效位是数字中有意义的位
  • Float的有效位至少32位，double至少48位，且不少于flaot，long double至少和double一样多。这三类的有效位可以一样多。指数范围至少是-37-37.

3.3.3. 浮点常量

  • 对于float类型添加后缀f或F，对于long double类添加后缀l或L
  • 对于float，C++只保留6位。
  • 浮点常量默认情况下为double。

3.3.4. 浮点数的优缺点

  • 优点：表示整数之间的值，存在缩放因子，可表示范围更大。
  • 缺点：运算速度慢，精度低。

C++算数运算符

3.4.1. 运算符的优先级和结合性

  • C++的结合性从左到右
  • C++的优先级自查

3.4.2. 除法分支

  • 对于运算符/，如果都为整数，C++将执行取整除法。如果其中有一个是小数，结果为浮点数。如果操作数都为float，结果为float。如果操作数都为double，结果为double。Long同理

3.4.3. 求模运算符

  • 当int相除时/为求模而%为求余数。

3.4.4. 类型转换

  • 将一种算术类型赋给另一种算术类型变量时，C++将对值进行转换
  • 表达式中包含不同类型时，C++将对值进行转换
  • 将参数传递给函数时，C++将对值进行转换。

3.4.4.1. 初始化和赋值进行的转换

  • C++将一种类型的值赋给另一种类型时，值将转换为接收变量的类型。这样就会存在一个问题，当将值赋给更小的类型时精度会降低，如将double转为float， 将浮点转为整型， 将long转化为short，这些做法都会带来麻烦。

3.4.4.2. 以{}方式初始化时进行转换

  • 列表初始化不允许缩窄，即变量的类型可能无法表示赋给它的值。编译器必须知道目标变量能够正确地存储赋给它地值。

3.4.4.3. 表达式中的转换

  • 一些值在出现时便会自动转换，如bool, char, unsigned char, signed char, 和short会直接转换为int。这些转换有个专业的名字叫做整型提升。
  • 在不同类型进行算术运算的时候也会转换，较小的类型会被转换为较大的类型。
  • 因此只存在两种情况，而int为这两种情况的分界点。
  • 除此之外再说明其他一些特殊情况。一个操作数有符号，另一个无符号，且无符号的级别高，则有符号的转换为无符号的。
  • 有符号能表示无符号的所有值，无符号的转换为有符号的。

3.4.4.4. 传递参数的转换

  • 通常情况下，传递参数依据函数原型，但是对char和short将会整型提升。Float也会提升为double。

3.4.4.5. 强制类型转换

  • (typename) value 或 typename value，不会修改value变量本身，而是创建一个新的，指定类型的值。
  • Static_cast<>强制将值从一种类型转换为另一种类型

3.4.5. C++中的auto

  • 根据初始值的类型推断变量的类型。
