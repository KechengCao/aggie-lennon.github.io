C++PrimerPlus-1

开始学习C++

2.1. 进入C++

2.1.1. 函数

  • C++对于大小写敏感，如错误地使用了大小写，编译器将指出使用未知地标识符。
  • 文件扩展名cpp是C++的常用方式。
  • 一个函数由四部分组成：函数头，在花括号中的函数体，函数返回类型和形参列表。函数头对函数与程序其他部分之间的接口做总结，函数体指出函数具体执行的计算指令。
  • C++的每条指令称为语句，以分号结束。

2.1.2. 注释

  • C++注释以//打头或者将注释放在/* 和 */之间。尽量使用前者。
  • 程序越复杂，注释的价值越大。

2.1.3. C++预处理器

  • 在源代码被编译前替换或添加文本（待具体）

2.1.4. 头文件名

  • 头文件：它们被包含在其他文件中，每个头文件都支持一组特定工具。
    头文件类型 约定 说明
    C++旧风格 以.h结尾 C++可用
    C旧风格 以.h结尾 C，C++可用
    C++新风格 无扩展名 C++可用，使用namespace std
    转换后的C 加c前缀,无扩展名 C++可用，可用不是C的特性，使用namespace std

    2.1.5. 名称空间

    • 在上表中的C++新风格和转换后的C都使用了名称空间using namespace std;
    • 使用名称空间的好处：在编写大型程序以及将多个厂商现有的代码组合起来的程序时更容易，有助于组织程序。对于同名的函数可通过不同的名称空间来区分。
    • Using编译啊指令和名称空间的关系：使用了using 编译指令的名称空间可是对应的名称空间中的所有名称都可用。
    • 比较推荐的做法using std;cout; 原因在于名称空间的管理方法。

    2.2. C++语句

    2.2.1. 声明语句和变量

    • 通过变量的声明告知了编译器两件事：需要的额内存和内存单元的名称。
    • 程序中的声明语句叫做定义声明，简称定义。而使用其他地方定义的变量称为引用声明。所以声明不一定是定义

    2.2.2. 赋值语句

    • 符号=为赋值运算符，C++允许使用连续赋值运算。

    2.3. 其他C++语句

    2.3.1. cin和cout

    • C++将输出和输入看作是流出和流入程序的字符流。cin使用>>从输入流中抽取字符，cout使用<<将字符输出到输出流。<<和>>用来表示信息流的方向。

    2.3.2. 类简介

    • 类是用户定义的一种数据结构。描述了表示什么信息和对数据能执行那些操作。
    • 类之于对象就像类型之于变量，类描述了一种数据类型的全部属性。

    2.4. 函数

    2.4.1. 有返回值的函数

    • 被调用的函数分为三个部分：函数的返回值赋值，函数头和函数体，传递的函数参数信息，和原型。
    • 被调用的函数称为调用函数。
    • C++程序应当为程序中使用的每个函数提供函数原型。函数原型之于函数就像变量声明之于变量。
    • 如何区分函数头和函数原型？函数原型的结尾有分号。

    2.4.2. 函数变体

    • 有些函数接受多个参数，有些函数不接受参数，还有一些函数没有返回值。对于C++，这些都叫做函数，无需区分为函数和过程。

    2.4.3. 用户定义的函数

    • 最简单的方法是将函数的定义放在main函数之前，将函数的的源代码放在main函数之后。

    2.4.4. 关键字

    • 像int, void, return, double等为C++的关键字，不能用作变量名，函数名和对象名。

    2.4.5. 在多函数程序中使用using编译指令

    • 将using指令放在函数定义之前，文件中的所有函数都能使用命名空间中的元素。
    • 将using指令放在特定函数定义之中，该函数能使用命名空间中的所有元素。
    • 在特定函数中使用类似using std::cout;这样的编译指令，该函数能使用命名空间中的指定元素。
    • 完全不适用using指令，需要使用命名空间中的元素是，使用前缀命名空间名称::
