复合类型

4.1. 数组

  • 数组是一种数据格式，能够存储多个同类型的值。
  • 数组声明应包括以下三点：存储在每个元素中的值的类型，数组名，数组中的元素数。
  • 数组声明通用格式typeneame arrayname[arraysize]
  • Arraysize必须是整型常数或const值，或者常量表达式。不能是变量。
  • 数组之所以被称为复合类型是因为其使用其他类型来创建。
  • 在使用下标访问数组中的元素时必须要确保下标值的有效性。

4.1.1. 数组的初始化规则

  • 只能在定义数组的时候初始化数组，以后就不能再初始化了，也不能将一个数组的值赋给另一个数组。
  • 初始化数组的时候提供的值可以少于数组元素的数目。剩余的元素将为0。
  • 如果初始化数组时中括号内为空，C++编译器将更具初始化的内容计算元素的个数。

4.1.2. C++11数组的初始化的方法

  • 列表初始化，可省略等号，可省略大括号中的内容
  • 列表初始化禁止缩窄转换

4.2. 字符串

  • 字符串是存储在内存中的连续字节中的一系列字符。
  • 有两种处理字符串的方法：C风格字符串和另一种基于string类库的方法。
  • C风格字符串是将字符串存放在char数组之中，以空字符结尾\0。这也是区分char数组是否是字符串的关键。在使用cout打印char数组时，答应过程会在遇到空字符串时停止。
  • 基于string类库的方式产生的字符串称为字符串常量或字符串字面值，这种字符串表示起来相对于C风格字符串更简洁。它会在结尾处隐式地加上空字符，因此，不必显示地添加。
  • 将字符串读入char数组时会自动加上空字符。因此在确定最短数组长度时别忘了预留空字符。
  • 字符串常量（双引号）和字符常量（单引号）不能互换。原因在于字符串常量多一个空字符并且值代表字符串所在的内存地址。

4.2.1. 拼接字符串常量

  • 任何两个由空格，制表符和换行符分隔的字符串常量都将自动拼接成一个。
  • 拼接字符串之间没有空格，上一个字符串的空字符将被下一个字符串第一个字符取代。

4.2.2. 在数组中使用字符串

  • 在数组中使用字符串的方法有两种：将一个数组初始化为用引号括起来的字符串，或使用cin将一个输入的字符串放入数组中。
  • Sizeof返回的是数组的长度。strlen返回的是数组中实际字符串的长度，不包括空字符。
    数组的长度至少是strlen+1。

4.2.3. 字符串输入

  • 在使用cin将用户输入的字符串存入char数组中时，cin将会使用空格，制表符和换行符来确定字符串结束的位置。这些符号会保留在输入队列之中。

4.2.4. 每次读取一行字符串输入

  • 那如何避免4.2.3中出现的问题呢？istream提供了一些面向行的函数:getline()和get()。
  • getline和get都将读取一行的输入，通过回车键输入的换行符来确定输入结尾。区别在于getline将丢弃换行符而get将保留换行符在输入序列之中。
  • getline函数有两个参数，数组名称和读取的额字符数，实际读取的比设定的少一，因为需要在结尾处添加空字符。遇到换行符时停止读取。
  • get函数如果不借助帮助将不能跨过换行符。解决方式是使用cin.get()来读取换行符。
  • get和getline都可以采用连续调用的额方式将两个类成员函数拼接起来。
  • get相比于getline的优势在于适应老式实现并且能通过get()知道读取因何停止。
  • 当读取到空行时，get的读取将中止。
  • 当输入字符串比分配空间长，多余的字符将留在输入队列之中，并且getline的读取将会停止。

4.3. string 类简介

  • 可以使用C风格字符串初始化string。可以使用cin来将键盘输入存储到string 对象。可以使用cout来显示string对象。可以使用数组表示法访问string对象中的字符。

4.3.1. C++字符串初始化

  • 对于C风格字符串和string对象字符串都可以使用列表初始化。

4.3.2. 赋值，拼接和附加

  • 对于string 对象可以将一个对象赋值给另一个对象。但对于C风格字符串数组就不支持。
  • String类可以用+合并。

4.3.3. string类的其他操作

  • 对于C风格字符串使用strcpy()将字符串赋值到字符数组中。使用strcat()将字符串附加到字符数组尾。当目标数组过小时可能会导致程序终止或者数据破坏。
  • 为了避免上述情况，strncat()和strncpy()接受指出目标数组最大允许长度的第三个参数。
  • C风格字符串中的strlen和string类中的size功能相似。

4.3.4. string类I/O

  • 对于没有初始化的C风格字符串，strlen的大小可能比sizeof大。
  • 对于C风格的字符串来说cin.getline(数组名，大小)，而对于string 类getline(cin, string名)。

4.3.5. 其他形式的字符串字面值

  • 对于wchar_t类型字符串字面值使用L前缀（放在引号外面）。对于char16_t使用u前缀。对于char32_t使用U前缀。
  • 对于原始字符串，字符表示的就是自己，所以无需转义，需要在加上R前缀。但(和)除外，使用R”+*(表示开头，使用)+*”表示结尾。

4.4. struct 类简介

  • Struct的组成部分：struct关键字， 标记成为新类型的名称，左右大括号， 结构成员， 结束结构声明。
  • 在C++中省略struct关键词不会出错。
  • 使用成员运算符.来访问各个成员。
  • 访问类成员函数的方式是从访问struct成员变量的方式衍生而来。
  • C++提倡使用外部结构声明。
  • struct的结构初始化使用都好分隔值列表，并将这些值用花括号括起。

4.4.1. C++11结构初始化

  • 支持列表初始化，等号可以忽略。花括号中可以不包含任何东西。

4.4.2. 其他结构属性

  • 可以将结构作为函数参数或者函数返回值。
  • 可以使用=将一个结构赋值给另一个结构，这种赋值方式称为成员赋值。

4.4.3. 结构数组

  • 初始化基金二狗数组的方式是使用一个被括在花括号中用逗号分隔的值列表，每个值又是被括在花括号中，每个结构成员的初始化各占一行。

4.5. 共用体

  • 共用体是一种数据结构，能存储不同的数据类型，但只能同时存储其中的一种类型。在存储另一种数据类型之后，原有的数据会丢失。
  • 必须有空间存储最大的成员。共用体的长度是其最大成员的长度。
  • 匿名共用体没有名称，其成员的地址相同，由程序员决定当前哪个成员是活动的。
  • 共用体的主要作用是节省内存。

4.6. 枚举

  • Enum的句法使用与结构的句法使用相似。
  • 默认情况下整数将被赋值给枚举量，从0开始。
  • 在没有强制转换的情况下，只能将定义时使用的枚举量赋值给枚举变量。
  • 枚举变量除了赋值运算不支持其他算数运算。
  • 枚举量能提升为整型int但是int不能变为枚举量。

4.6.1. 设置枚举量的值

  • 使用赋值运算符显示地设置，但值是int,long,甚至时long long。没设置的值从设置的值开始+1递增。

4.6.2. 枚举量的取值范围

  • 每个枚举都有取值范围，通过强制类型转换，可将取值范围内的任何整数赋给枚举变量，即使不是枚举值。上限是大于最大值的最小2的幂，然后-1。对于下限的寻找分两种情况：不小于0则下限直接是0， 小于0是小于最小值的最大2的幂，然后+1。
  • 对于C++11，存在作用域内枚举。

4.7. 指针和自由存储空间

  • 指针是一个变量，其存储的是值的地址。通过在变量前面加上&来获取变量的地址。
  • cout使用16进制来显示出变量的地址。
  • *运算被称为间接值或者解除引用运算符

4.7.1. 声明和初始化指针

  • 由于不同的数据类型存储时使用的内部格式不同，因此，声明指针必须指定指针指向的数据类型。
  • 对于每个指针变量名都需要使用一个*。
  • 指针的大小取决于存储一个地址需要的大小，而地址的大小可为2或者4，这主要看计算机系统。

4.7.2. 指针的危险

  • 指针需要被初始化，不然可能为任何值，这将导致一些很隐匿的bug。

4.7.3. 指针和数字

  • 对于C++来说，如果想使用数字值作为地址使用，应通过强制类型转换将数字转换为适当的地址类型。

4.7.4. 使用new来分配内存

  • 指针的很常用的做法是在运行阶段分配未命名的内存以存储值。那这种未命名的内存通过使用new来分配内存。通用格式为 typename * pointer_name = new typename。
  • 与常用的额变量声明分配时使用栈不同，new从被称为堆或者自由存储区的内存区域分配内存。

4.7.5. 使用delete释放内存

  • 使用delete运算符，在使用完内存后将其归还给内存池。通用写法时delete pointer_name。
  • 使用delete不会删除指针本身，指针可以再利用。
  • 需要注意的一点是new和delete必须配对使用，否则会导致内存泄漏。
  • 不要将两个指针分配给同一个内存块，因为这违反了new和delete需要配对的原则。

4.7.6. 使用new来创建动态数组

  • 在编译时分配内存被称为静态联编，但使用new能在程序运行的时候分配内存，这被称为动态联编。使用动态联编创建出来的数组称为动态数组。
  • 创建动态数组的通用格式type_name * pointer_name = newtype_name[num_elements];其中new运算符返回的是第一个元素地址。
  • 释放动态数组内存的格式为delete [] pointer_name；
  • 使用new和delete时因遵守以下规则：不要使用delete释放不是new分配的内存，不要使用delete释放同一个内存块两次，new[]对应delete[]，new无方括号delete也没有方括号，对空指针使用delete是安全的。
  • 对于动态数组的大小，这种数据不是公用的，例如，不能使用sizeof来获取动态分配的数组包含的字节数。
  • 那么如何使用动态数组呢？通过将动态数组的指针当作数组名来使用。可通过加上整数来调整指针在数组中的指向。

4.8. 指针，数组和指针算术

  • 从上面我们看到C++对于指针的使用和数组的使用如此相似，那背后的原因在于指针算术。Arrayname[i] == *(arrayname + i)
  • 这里有一点需要注意，对于数组名是第一个元素的地址而数组名加上前缀&表示的是整个数组的地址。Array+1 != &array + 1。
  • 需要注意的是typename pointer_name[size] = &arrayname与typename （pointer_name）[size] = &arrayname不同。前者是一个typename指针数组包含size个元素，而后者是一个指针指向size个typename。

4.8.1. 指针小结

  • 声明指针格式：typename * pointername;
  • 给指针赋值分两种：&来获得被命名的内存地址，new运算符返回未命名的内存地址。
  • 对指针使用*或者数组表示法pointer_name[0]来解除引用。
  • C++将数组名视为数组的第1个元素地址。
  • 声明数组创建数组为静态编译，使用new[]创建数组为动态编译。
  • 数组表示法和指针表示法胡同arrayname[index] = *(arrayname + index)。

4.8.2. 指针和字符串

  • 在C++的多数表达式中，char数组名，char指针和引号阔气的字符串常量都可解释为字符串的第一个字符地址。

4.8.3. 使用new创建动态结构

  • 在创建时需要同时使用结构类型和new
  • 由于使用了new，结构未命名，不能使用运算符(.)来访问成员，一种替代方法是使用->

4.8.4. 自动存储，静态存储和动态存储

  • 自动存储：常规变量使用自动存储空间，也被称为自动变量，在其所在函数调用时产生，结束时消亡，其通常存储在栈中，后进先出。内存为连续的。
  • 静态存储：定义静态变量的方式有两种：在函数外定义或加上关键词static
  • 动态存储：它们管理着一个内存池也被称为自由存储空间或堆，堆和静态存储，自动存储的内存分开。内存不连续。

4.9. 数组的替代品

4.9.1. 模板类vector

  • Vector类似于string类，是一种动态数组。创建的通用表达式vector vt(n_elem)。

4.9.2. 模板类array

  • 相比于vector，array的效率高，但代价是不方便和不安全

4.9.3. 对比数组，array和vector

  • Array存在于栈而vector对象存储在堆中
  • 数组必须逐个元素复制而array和vector可以直接将一个对象赋值到一个对象。
  • 使用array和vector的时候可以使用成员函数at()来捕获运行时的非法索引，但代价时运行时间更长。
